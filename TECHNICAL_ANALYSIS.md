## 1. КРИТИЧЕСКИЕ ПРОБЛЕМЫ БЕЗОПАСНОСТИ

### 1.1 Недостаточная валидация и санитизация данных
**Где:** Все компоненты, получающие данные из API  
**Проблема:** 
- Нет валидации структуры данных, приходящей с сервера
- Отсутствует проверка на XSS-атаки при отображении пользовательского контента
- Данные из localStorage парсятся без try-catch блока

**Последствия:** Потенциальные XSS-атаки, краш приложения при некорректных данных  
**Файлы:** 
- `src/redux/slices/data/actions.ts`
- `src/utils/getCartFromLS.ts`
- `src/components/Catalog/CatalogItem/index.tsx`
- `src/pages/FullPizza/index.tsx`

---

### 1.2 Hardcoded API endpoint в FullPizza
**Где:** `src/pages/FullPizza/index.tsx` (строка 15)  
**Проблема:**
```tsx
const API = "https://react-pizza-api.up.railway.app/api"; // ❌ Hardcoded
```
- API URL жестко закодирован, не используется переменная окружения
- В actions.ts используется `import.meta.env.VITE_API_URL`, а здесь - нет
- Разный подход к получению API между компонентами

**Последствия:** Невозможно переключаться между окружениями, проблемы при развертывании  
**Исправление:** Использовать единую точку конфигурации для API

---

### 1.3 Отсутствие обработки ошибок при API запросах
**Где:** 
- `src/redux/slices/data/actions.ts` (fetchPizza, fetchPizzaById)
- `src/pages/FullPizza/index.tsx`

**Проблема:**
```ts
// ❌ Ошибка просто возвращается, без обработки
catch (error) {
  return rejectWithValue(error);
}

// ❌ В FullPizza используется alert()
catch (error) {
  alert("УВЫ. Пицца не найдена D:");
  navigate("/");
}
```

- Нет информации об ошибке для логирования
- Используется `alert()` - плохая UX
- Нет различия между типами ошибок (сетевая, 404, 500)
- Отсутствует retry логика

**Последствия:** Пользователь не понимает, что произошло; сложная отладка  
**Файлы:** `src/redux/slices/data/actions.ts`, `src/pages/FullPizza/index.tsx`

---

### 1.4 Отсутствие обработки недоступности localStorage
**Где:** `src/utils/getCartFromLS.ts`, `src/components/Header/index.tsx`  
**Проблема:**
```ts
// ❌ Нет проверки доступности localStorage
const storage = localStorage.getItem("user_cart");

// ❌ Может выбросить ошибку в приватном режиме браузера
localStorage.setItem("user_cart", json);
```

- В приватном режиме браузера или при отключении хранилища - ошибка
- Нет fallback механизма

**Последствия:** Краш приложения в приватном режиме браузера  
**Файлы:** `src/utils/getCartFromLS.ts`, `src/components/Header/index.tsx`

## 2. УЯЗВИМОСТИ СОКРАЩЕНИЯ URL

### 2.1 SQL-injection подобная уязвимость при построении URL
**Где:** `src/redux/slices/data/actions.ts` (строка 12-14)  
**Проблема:**
```ts
// ❌ URL строится конкатенацией без экранирования
const { data } = await axios.get(
  `${API}/pizza?page=${+pageProperty + 1}&limit=4&search=${searchValue}&${sortProperty}${filterProperty}`
);
```

- `searchValue` может содержать спецсимволы, которые нарушат URL
- `filterProperty` и `sortProperty` строятся вручную из slice
- Нет валидации входных параметров
- Потенциально возможна инъекция параметров

**Последствия:** Непредсказуемое поведение API, возможные инъекции  
**Исправление:** Использовать URLSearchParams или объект параметров axios

---

## 3. ПРОБЛЕМЫ С УПРАВЛЕНИЕМ СОСТОЯНИЕМ И ПАМЯТЬЮ

### 3.1 Утечка памяти при подписке на Redux в Header
**Где:** `src/components/Header/index.tsx` (lines 15-23)  
**Проблема:**
```tsx
const isMounted = useRef(false);

useEffect(() => {
  if (isMounted.current) {
    // Сохранение в localStorage
  }
  isMounted.current = true;
}, [data]); // ❌ Отсутствует cleanup функция
```

- `useEffect` не имеет cleanup функции
- `localStorage.setItem` вызывается на каждое изменение data
- Нет защиты от множественных запросов

**Последствия:** Лишние операции localStorage, потенциальные утечки памяти  
**Исправление:** Добавить cleanup функцию и дебаунс

---

### 3.2 Отсутствие очистки дебаунса при демонтировании компонента
**Где:** `src/components/Search/index.tsx`  
**Проблема:**
```tsx
useEffect(() => {
  return () => debouncedSearch.cancel?.(); // ✓ Хорошо, но может быть better
}, [debouncedSearch]);
```

- Хотя cleanup добавлен, `debouncedSearch` пересоздается при каждом `dispatch`
- Излишняя пересоздание debounce функции в useMemo

**Последствия:** Дополнительные выделения памяти  
**Исправление:** Переместить dispatch в зависимости более оптимально

---

### 3.3 Проблема с ключами в map() функциях
**Где:** `src/components/CartCatalog/index.tsx` (line 11)  
**Проблема:**
```tsx
// ❌ Использование индекса как ключа
{data.map((item, index) => (
  <CartItem
    key={index}  // ❌ Плохо! Переупорядочение сломает компонент
    ...
  />
))}
```

**Последствия:** При удалении/добавлении товаров в корзину возможны баги, неправильное состояние компонентов  
**Исправление:** Использовать уникальный идентификатор (`id`)

---

### 3.4 Проблема с состоянием страницы при изменении фильтров
**Где:** `src/pages/Home/index.tsx`  
**Проблема:**
```ts
// ❌ При изменении фильтра остается старая страница
useEffect(() => {
  dispatch(fetchPizza({
    sortProperty,
    filterProperty,
    pageProperty: String(pageProperty), // Может быть не 0
    searchValue,
  }));
}, [dispatch, sortProperty, filterProperty, pageProperty, searchValue]);
```

- Когда пользователь меняет категорию/сортировку, остается старая страница
- Например, если был на странице 3, и выбрал другую категорию, может быть 404
- Отсутствует автоматический сброс на страницу 0

**Последствия:** Пустой каталог при переходе на несуществующую страницу  
**Исправление:** Автоматически сбрасывать страницу при изменении фильтров

## 5. ПРОБЛЕМЫ С ОПТИМИЗАЦИЕЙ ПРОИЗВОДИТЕЛЬНОСТИ

### 5.1 Отсутствие мемоизации компонентов
**Где:** Несколько компонентов  
**Проблема:**
```tsx
// ❌ Компоненты без memo перерендериваются при каждом обновлении родителя
export const CatalogItem: React.FC<IProps> = ({ ... }) => { ... }
export const Categories: React.FC = () => { ... }
export const Sort: React.FC = () => { ... }
```

- `CatalogItem` перерендеривается при каждом обновлении каталога
- `Categories`, `Sort` перерендериваются часто

**Последствия:** Лишние рендеры, замедление приложения при большом каталоге  
**Исправление:** Обернуть в `React.memo()`

---

### 5.2 Неэффективные селекторы Redux
**Где:** `src/redux/slices/cart/selectors.ts`  
**Проблема:**
```ts
export const selectCartById = (id: string) => (state: RootState) =>
  state.cart.data.find((item) => item.id === id); // ❌ O(n) поиск
```

- Каждый вызов `selectCartById` создает новую функцию-селектор
- В `CatalogItem` вызывается для каждого товара
- Используется `.find()`, что O(n) операция

**Последствия:** На каталоге с 100+ товарами может быть значительное замедление  
**Исправление:** Нормализовать данные в Redux или использовать Map/объект

---

### 5.3 Неоптимальные зависимости в useEffect
**Где:** `src/pages/Home/index.tsx`  
**Проблема:**
```ts
useEffect(() => {
  dispatch(fetchPizza({...}));
}, [dispatch, sortProperty, filterProperty, pageProperty, searchValue]); // ❌ Много зависимостей
```

- При каждом изменении любого из параметров - новый запрос
- Но sortProperty и filterProperty могут быть объектами, которые пересоздаются

**Последствия:** Лишние API запросы  
**Исправление:** Мемоизировать параметры или использовать более точные зависимости

---

### 5.4 Отсутствие виртуализации большого списка
**Где:** `src/components/CartCatalog/index.tsx`  
**Проблема:**
- При большой корзине (100+ товаров) все компоненты CartItem рендерятся одновременно
- Нет виртуализации или ленивой загрузки

**Последствия:** Медленная скорость при большой корзине  
**Исправление:** Использовать react-window или react-virtualized

---

### 5.5 Неоптимальное использование PizzaLoader
**Где:** `src/components/Catalog/index.tsx` (line 24)  
**Проблема:**
```tsx
{..Array(4)].map((_, index) => <PizzaLoader key={index} />)}
```

- Создается 4 лоадера, но это может не соответствовать реальному количеству элементов
- Все лоадеры показывают одинаковую ширину, что может не совпадать с сеткой

**Последствия:** Дёргание интерфейса при загрузке  
**Исправление:** Сделать адаптивное количество лоадеров в зависимости от размера
